# 产品需求文档 (PRD)：AI 赋能企业供应链可视化分析系统 (MVP 版)

## 1. 项目概述

本项目旨在开发一套轻量级的供应链可视化与智能预警系统。鉴于开发周期为一个月，本阶段将不直接对接企业真实业务系统，而是采用 Kaggle 开源供应链数据集，通过构建“数据回放引擎”模拟实时高并发数据流，结合 AI 算法实现对供应链链路状态的动态监控与异常预警。

## 2. 核心目标 (MVP)

* **数据流动可见：** 实现静态数据集的时序化回放，前端大屏能实时展示动态刷新的订单流与物流状态。
* **智能风险阻断：** 建立基于机器学习的风险预警模型，毫秒级识别异常订单并主动推送告警。
* **闭环演示交付：** 形成“数据产生 -> 后端接收 -> AI 判定 -> 前端预警”的完整闭环，支撑最终的项目演示与答辩。

## 3. 用户角色与核心场景

* **角色：** 供应链风控专员 / 管理层
* **核心场景：** 用户打开系统大屏，看到当天的核心指标（总单量、准交率）在实时跳动。突然，系统底层拦截到一条高风险的模拟订单数据，大屏右侧的“风险事件追踪”列表顶部立即闪烁红色高亮，提示某笔重要物资存在“极高延迟交付风险”，并展示预测依据。

## 4. 业务功能需求

### 4.1 数据工程与流处理模块

| 功能名称 | 需求描述 | 优先级 |
| --- | --- | --- |
| 数据清洗与特征提取 | 对选定的 Kaggle 数据集进行离线预处理，提取时间戳、地理位置、物料类别等核心字段。 | P0 |
| 实时数据回放引擎 | 编写独立后台脚本，按设定倍率（如1秒=真实世界1小时）逐条读取清洗后的时间序列数据，并通过 HTTP/接口发送至后端。 | P0 |

### 4.2 AI 智能预警模块

| 功能名称 | 需求描述 | 优先级 |
| --- | --- | --- |
| 基准风控模型训练 | 利用历史数据训练二分类模型（如 Random Forest 或 XGBoost），预测目标为“是否延迟交货”或“是否存在欺诈风险”。 | P0 |
| 实时在线推理服务 | 后端接收到单条“实时”数据后，立即调用加载好的模型文件进行推理，输出风险概率阈值。 | P0 |

### 4.3 可视化监控大屏模块 (前端 UI)

| 功能名称 | 需求描述 | 优先级 |
| --- | --- | --- |
| 全局核心指标盘 | 展示累计订单量、实时 GMV、风险拦截次数（通过 WebSocket 实时跳动更新）。 | P0 |
| 物流热力地图 | 基于 ECharts 展示全国/全球物流节点路线连线，支持动态光点表示运输中的货物。 | P1 |
| 实时预警滚动播报 | 以滚动列表形式展示被 AI 拦截的异常订单详细信息（单号、风险类型、拦截时间），高风险采用醒目颜色标注。 | P0 |
| 历史趋势折线图 | 展示过去 N 小时/天的进出货量波动情况。 | P2 |

## 5. 非功能需求与技术规范

### 5.1 技术栈选型建议

* **前端：** Vue 3 + Vite + ECharts + WebSocket 客户端
* **后端：** Python (FastAPI - 具备极高的并发性能且天生契合 Python AI 生态)
* **数据存储：** MySQL (结构化业务数据) + Redis (实时缓存与 WebSocket 会话状态)
* **算法库：** Scikit-learn / Pandas
* **部署：** Docker Compose 一键容器化编排

### 5.2 性能与交互要求

* **实时性：** 从数据发生器发出数据，到前端大屏接收到 WebSocket 告警刷新，端到端延迟控制在 1 秒以内。
* **前端性能：** 数据降采样处理，保证大屏长时间挂机展示时不发生内存溢出（OOM）或严重掉帧卡顿。

## 6. 数据流向与接口定义初步

1. **`[离线]` AI 模型产出：** `.csv` 数据 -> Python Jupyter 训练 -> 导出 `risk_model.pkl`。
2. **`[实时]` 数据注入接口：**
* `POST /api/v1/stream/ingest`
* 接收 JSON 格式的单条订单/物流节点数据。


3. **`[实时]` WebSocket 推送通道：**
* `ws://domain/api/v1/ws/alerts`
* 当后端拦截到风险概率 > 85% 的单据时，主动向该通道广播 JSON 告警信息。



## 7. 最终验收与演示剧本 (Demo Script)

1. **环境初始化：** `docker-compose up -d` 拉起所有服务。前端展示空白（或初始状态）的大屏。
2. **启动引擎：** 后台终端执行 `python data_producer.py`。
3. **动态展示：** 评委/观众看到前端大屏各项数字开始规律性跳动，折线图开始向右推演。
4. **触发高潮：** 脚本运行到预埋的“异常黑天鹅数据”节点，AI 模型瞬间捕捉。
5. **视觉冲击：** 大屏 UI 发生变化，弹出警告框“发现编号 [xxx] 订单存在欺诈风险，系统已自动拦截”，演示完美结束。
